## 4. Module Boundary Map

### 4.1 Modules to Create (NEW)

| Module | Path | Responsibility |
|--------|------|----------------|
| `ml/knowledge/base.py` | Knowledge Base Engine | Commodity record definitions, query interface, validation |
| `ml/recommendation/rules.py` | Rule Engine | Inference rules, risk mapping, decision set generation |
| `ml/recommendation/explain_v2.py` | Explanation Engine v2 | Rule-chain-based explanations |

### 4.2 Modules to Replace

| Current Module | Replaced By | Rationale |
|----------------|-------------|-----------|
| `ml/recommendation/scorer.py` | `ml/knowledge/base.py` + `ml/recommendation/rules.py` | Heuristic scoring replaced by rule-based inference |
| `ml/recommendation/recommender.py` | `ml/recommendation/rules.py` | Top-N ranking replaced by category-based recommendation |
| `ml/recommendation/explain.py` | `ml/recommendation/explain_v2.py` | Explanations now reference rule chain, not score components |

### 4.3 Modules to Modify

| Module | Modification |
|--------|--------------|
| `ml/recommendation/mitigation.py` | Add per-commodity mitigation coupling; accept decision set context |
| `ml/service/predictor.py` | Replace recommend() call with kb_recommend() call; enrich output |
| `backend/schemas/response.py` | Add new response fields (knowledge_source, alternative, not_recommended) |
| `backend/routers/realtime.py` | Wire new response fields into API output |

### 4.4 Modules Remaining Unchanged

| Module | Path |
|--------|------|
| Random Forest Predictor | `ml/predict/random_forest.py` |
| LSTM Predictor | `ml/predict/lstm.py` |
| Risk Classifier | `ml/predict/risk.py` |
| Feature Engineering | `ml/feature_engineering/` |
| Prediction Preprocessing | `ml/predict/preprocess.py` |
| OpenMeteo Provider | `backend/providers/openmeteo_provider.py` |
| Prediction Gateway | `backend/services/prediction_gateway.py` |
| Auth Dependencies | `backend/dependencies/auth.py` |
| Rate Limiting | `backend/security/` |
| Weather Providers | `backend/providers/` |

## 5. Configuration Management

The KB-DSS recommendation layer will be configurable via:

```yaml
# ml/config/kb_config.yaml
knowledge_base:
  version: "1.0"
  profile_source: "ml/knowledge/commodity_profiles.json"
  validation: strict

rule_engine:
  version: "1.0"
  risk_thresholds:
    rendah_max: 33.0
    sedang_max: 66.0
  enable_duration_penalty: true
  enable_economic_weighting: true

output:
  include_reasoning_chain: true
  max_recommended: 5
  max_alternative: 3
  max_not_recommended: 5
```

## 6. Error Handling Strategy

| Scenario | Behaviour |
|----------|-----------|
| Knowledge base load failure | System falls back to static default recommendations; logs error |
| Unknown commodity ID in rule | Rule engine skips unknown ID; logs warning |
| Rule engine exception | Returns empty recommendation set with error flag |
| Missing FRI value | Rule engine returns "insufficient data" status |
| Invalid risk level | Defaults to "Rendah" with logged warning |
